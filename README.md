Написанный алгоритм, реализованный на языке Python, был визуализирован с помощью языка react, серевер разворачивается с помощью сервиса ngrok.
На вход подаётся таблица csv в формате "ID", "Начало", "Длительность", "Окончание", "Последователи","Предшественники" (Разделитель - ;).
По данным из таблицы по номерам последователей после преобразования в dataframe строится граф, который отражает все зависимости между задачами. При изменении времени исполнения задачи или ее длительности граф перестраисвается, следуя новым зависимостям.   
При этом считается стоимость переноса или удлинения задачи. Результатом является перестроенный граф, представимый в виде csv таблицы, и количество затрат при переносе. 

На данный момент поддерживается только связь ОН, другие связи между задачами (НО, ОО, НН) игнорируются.

В data представлены тренировочные таблицы, проверка которых происходила при отладке.

Алгоритм работает только на таблицах того же вида, что и представленная организаторами. Однако в данной таблице присутствуют такие случаи, что дата окончания задачи-последователя раньше, чем дата окончания задачи-предшественника (1703 и 4), но связи при этом не указаны, и таких случаев несколько.
Из-за этого при построении графа алгоритм сразу перестраивает таблицу и выдает крупные значения стоимости переноса работ.
